表达式和语句

表达式

表达式（expression）是JavaScript中的一个短语，JavaScript解释器会将其计算出一个值。
一个表达式可以产生一个值
表达式可以放在任何需要值的地方。
表达式可以是：数据 或者 数据和运算符的组合
例如：1、’abc’、true、1 + 2、i++、a === b ? a : b 、a = 1

只要是能够作为 方法 参数的内容，就是： 表达式
console.log( 表达式 );

语句

语句可以理解为一个行为，循环语句和判断语句就是典型的语句。
一个程序有很多个语句组成，一般情况下用;分割一个一个的语句

continue 、 break 、return 的比较

break : 能够在：循环、switch-case 中使用
立即跳出整个循环，即循环结束，开始执行循环后面的内容
continue : 在 循环 中使用
立即跳出当前循环，继续下一次循环（跳到i++的地方）
return : 也需要区分开，return 只能在函数中使用！
作用：让当前函数停止运行！为函数提供返回值！

注释

不被程序执行的代码。用于程序员标记代码，在后期的修改，以及他人的学习时有所帮助，在JS中，分为单行注释和多行注释

单行注释

用来描述下面一个或多行代码的作用

多行注释

用来注释多条代码

注释的作用：

1. 模块划分，方便代码查找和维护
2. 用于解释复杂代码的逻辑，方便维护和后期开发。

要求：写代码的时候必须要写注释

代码调试

  • 过去调试JavaScript的方式
    • alert() 弹框警告–不常用
    • console.log() 控制台输出日志
    • document.write 往页面中写入内容
    • confirm 确认框–不常用
    • prompt 输入框–不常用
  • 断点调试

断点调试是指自己在程序的某一行设置一个断点，调试时，程序运行到这一行就会停住，然后你可以一步一步往下调试，调试过程中可以看各个变量当前的值，出错的话，调试到出错的代码行即显示错误，停下。

tips: 监视变量，不要监视表达式，因为监视了表达式，那么这个表达式也会执行。

  1. 代码调试的能力非常重要，只有学会了代码调试，才能学会自己解决bug的能力。初学者不要觉得调试代码麻烦就不去调试，知识点花点功夫肯定学的会，但是代码调试这个东西，自己不去练，永远都学不会。

异常处理

前面代码出现了错误，会影响到后续代码的执行！为了防止代码可能的出错导致后续代码的不执行，需要用到异常处理。
