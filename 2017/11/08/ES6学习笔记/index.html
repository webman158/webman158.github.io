箭头函数

  1. 箭头函数没有函数声明
  2. 箭头函数要吗是函数表达式要是匿名函数
  3. 箭头函数不能当做构造函数，也就是不能 new，否则会报错
  4. 箭头函数内部不要使用arguments对象，会有问题，可以使用rest参数代替arguments。
  5. 关于箭头函数this的指向问题
    • 可以理解为箭头函数相当于默认了bind了当前执行环境的this指向。
    • 也可以理解为箭头函数内部没有自己的this，所以去外部查找this 的指向问题。

6.使用箭头函数返回一个对象，对象必须要用小括号包裹起来

let和const比较

const 是不变的，分为两种情况：

  1. 值类型
  2. 引用类型，对于引用类型，不可以更改引用，但是可以修改内部成员的取值

const 声明的时候一定要赋值，否则会报错！

let 不存在变量提升的问题以及不可以重复声明

意外的发现

总结：
(这句话是在上面样的情况下得到验证的？)

  1. 凡是变量使用 let
    凡是不变的都是用 const

如果定义成员，一次都没有改过，使用const
如果定义的成员，最少做了一次修改，则一定是let

所以相对来说，使用const要多一些
最佳使用建议:全部都使用const，当报错的时候改为let

  1. const、let都是有块级作用域的

class关键字

  • ES6以前，JS是没有class概念的，而是通过构造函数+原型的方式来实现的
  • 注意：ES6中的class仅仅是一个语法糖，并不是真正的类，与Java等服务端语言中的类是有区别的

  1. 类必须使用new调用，否则会报错。这是它跟普通构造函数的一个主要区别，后者不用new也可以执行。

  2. 通过类内部的 constructor() 函数来给实例对象添加属性
    注意： constructor 是固定的名称

  3. 通过类内部的自定义方法，给实例对象添加实例方法
    注意：这种方式添加的实例方法是添加到 原型对象 中的
