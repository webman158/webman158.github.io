webpack的两种安装方式

  1. 运行npm i webpack -g全局安装webpack，这样就能在全局使用webpack的命令
  2. 在项目根目录中运行npm i webpack --save-dev安装到项目依赖中(推荐)

  1. src 目录中，存放着项目开发阶段的源代码
  2. dist 目录中，存放的是项目发布完成之后，可以直接上线的代码

实现webpack的实时打包构建

  1. 由于每次重新修改代码之后，都需要手动运行webpack打包的命令，比较麻烦，所以使用webpack-dev-server来实现代码实时打包编译，当修改代码之后，会自动进行打包构建。
  2. 运行npm i webpack-dev-server --save-dev安装到开发依赖
  3. 在package.json文件中添加

运行的时候直接npm run dev就可以实时打包构建了。但是这时候没有生成打包文件，因为webpack-dev-server命令是将打包好的文件放在内存中了

  • webpack-dev-server 这个工具在启动的时候，会先按照 webpack.config.js 中的配置规则，将模块打包到内存中，然后启动一个 Server 服务器运行当前的项目，同时自动监视了所有的资源，当资源发生改变，然后自动刷新你的浏览器.所以在项目目录里找不到打包好的文件。
  • 把bundle.js放在内存中的好处是：由于需要实时打包编译，所以放在内存中速度会非常快
