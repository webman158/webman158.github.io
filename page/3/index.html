表达式

只要是能够作为 方法 参数的内容，就是： 表达式

什么是JS语言

JavaScript是一种运行在客户端 的脚本语言
客户端：即接受服务的一端，与服务端相对应，在前端开发中，通常客户端指的就是浏览器。
脚本语言：也叫解释型语言，特点是执行一行，解释一行，如果发现报错，代码就停止执行。
JavaScript的解释器被称为JavaScript引擎，为浏览器的一部分，广泛用于客户端的脚本语言，最早是在HTML（标准通用标记语言下的一个应用）网页上使用，用来给HTML网页增加动态功能。

JavaScript最初的目的

最初的目的是为了处理表单的验证操作。

JavaScript现在的意义(应用场景)

JavaScript 发展到现在几乎无所不能。

  1. 网页特效
  2. 服务端开发(Node.js)
  3. 命令行工具(Node.js)
  4. 桌面程序(Electron)
  5. App(Cordova)
  6. 控制硬件-物联网(Ruff)
  7. 游戏开发(cocos2d-js)

vue初学笔记整理

Veröffentlicht am 2017-11-09

前置：

  1. 再用vue-cli组件式的运用vue，在main.js里面初始化vue实例的时候，渲染被替换的#app节点只能用render函数的原因

总结：

  1. 注意查看vue不同版本的区别：[对不同构建版本的解释]https://cn.vuejs.org/v2/guide/installation.html#%E5%AF%B9%E4%B8%8D%E5%90%8C%E6%9E%84%E5%BB%BA%E7%89%88%E6%9C%AC%E7%9A%84%E8%A7%A3%E9%87%8A ()
  2. 同时对应的查看node-modules/vue/package.json文件中不同加载方式对应的加载到的vue版本
  3. render 函数的作用
  4. 使用ES6模块来加载vue，如果实在非要使用vue.esm.js版本的话，不建议更改vue目录下的package.json文件中的module属性来更改加载vue.esm.js版本。因为这样将项目给别人，自己的更改将不能同步，导致在别人电脑上运行时还是报错的信息。这时候可以使用webpack的reslove选项来配置vue的别名

类似的，vue-cli脚手架做的'@': resolve('src')事情是一样

  1. Vue 选项中的 render 函数若存在，则 Vue 构造函数不会从 template 选项或通过 el 选项指定的挂载元素中提取出的 HTML 模板编译渲染函数。
